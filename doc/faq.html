1.1. What is the purpose of this file?
This file is an attempt to present and answer some of most frequently asked questions about Majik MURPE. It is advised that everyone interested in Majik should make him/herself familiar with this file thus avoiding the need for the developers of Majik to answer the questions herein. This file is written by members of the Majik core development team so everything presented here is absolutely reliable and is the official view of the Majik development team.

1.2. What does "FAQ" stand for?
"FAQ" stands for Frequently Asked Questions.

1.3. Who is the maintainer of this file?
This file is maintained by the members of the Majik4 core development team. The file was initially created by Yorkaturr (yorkaturr@majik.netti.fi) and converted to HTML by Namhas (namhas@majik.netti.fi).

2.1. What is Majik?
Majik is an Internet based computer game. It is a mixture of MUDs, Roguelikes and role-playing/adventure games such as Ultima. All of these games originate from pencil and paper -based roleplaying games such as AD&D, Rolemaster and Rune Quest and Majik hasn't forgotten it's roots.

2.2. What is a MURPE?
MURPE stands for Multi-User Role-Playing Environment.

2.3. What is Majik4?
Majik 4 is the fourth distinguishable incarnation of Majik. All of Majik's predecessors have been essentially MUDs but Majik4 combines the old Majik elements with new Ultima and Roguelike -type elements both technically and in game-terms. What this means in practice is that Majik4 will feature an external client program which allows us to use an intuitive ASCII-based tile-graphics system (we call it "gridmap") similar to that in Roguelike-games, especially ADOM, while featuring a key shortcut -based user interface as well as a prompt-based input system similar to that in normal telnet-based MUDs. At the moment Majik4 is under development and only open to the development team of Majik.

2.4. When will Majik4 be open to the public?
When the base code is ready. Should be fairly soon.

3.1. Why a separate client?
In the times of Majik3 we tried to create a gridmapped environment without any external programs and faced many difficulties. Now we are able to do great deal of the most intensive calculations in the client reducing server load and make the user-interface much more friendlier.

3.2. Where can I get the client?
The client is not available for public download until the base code has been finished. It will be available from Majik homepages when the time comes due.

3.3. What platforms does the client support?
The client is primarily developed under Linux (ncurses, GTK), which is currently the only platform supported. We will make at least version for Windows (95/NT), compile it for the most popular unixes and for Amiga of course. Also, other ports might be done in the future (java, for example).

3.4. Is the client source available?
No. We won't release the source because someone could alter the client to display more information than he/she is supposed to see.
